## Lessons learned

- **Legacy field renames need a triple sweep:** struct definition, struct literal call-sites in tests, AND any JSON/SQLite overlay shims. Grepping only the production package missed `tests/release_test/`.
- **`go vet` non-constant format string** triggers when `fmt.Fprintf` receives a dynamic string as the format arg — fix is to use `fmt.Fprint` or pass `"%s"` explicitly.
- **Compat shims are cheap to keep but expensive to remove silently** — a decision record (date + 5-step removal plan + version target) prevents the next AI from second-guessing and ripping them out prematurely.
- **ERD drift is invisible without a test** — the v3.5.0 ERD was missing 11 tables and nobody noticed; the proposed `SQLCreate*` parity CI test would have caught this.
