# Solved Issues

## v2.74.0

### 01 — Setup Config Path Resolution
- **Symptom**: `gitmap setup` failed with `open ./data/git-setup.json: The system cannot find the path specified`
- **Root Cause**: Config path was resolved relative to CWD instead of the binary's installed location
- **Solution**: Added `resolveSetupConfigPath()` that uses `filepath.EvalSymlinks(os.Executable())` to anchor the config path to the binary directory
- **Files Changed**: `cmd/setup.go`, `cmd/doctorsetup.go`, `constants/constants_doctor.go`
- **Learning**: Always resolve data file paths relative to the binary, never CWD
- **Never Repeat**: Do not use `./data/` relative paths for files bundled with the binary

### 02 — Shell Wrapper Not Detected
- **Symptom**: `gitmap cd` silently failed — changed directory in child process but parent shell stayed put
- **Root Cause**: No mechanism to distinguish raw binary invocation from shell wrapper invocation
- **Solution**: Shell scripts now export `GITMAP_WRAPPER=1`; `gitmap cd` checks this env var and prints stderr warning with `gitmap setup` instructions when missing
- **Files Changed**: Shell wrapper templates, `cmd/cd.go`, `cmd/doctorsetup.go`
- **Learning**: Shell-dependent commands MUST verify the wrapper is active before running
- **Never Repeat**: Do not add commands that require parent shell interaction without wrapper detection

## v2.75.0

### 03 — Clone-Next Flatten Implementation
- **Symptom**: Users accumulated many versioned folders (`macro-ahk-v11/`, `macro-ahk-v12/`, etc.)
- **Root Cause**: `clone-next` created a new folder for each version iteration
- **Solution**: Changed default behavior to clone into base name folder (no `-vN` suffix), auto-remove existing folder, track version transitions in `RepoVersionHistory` DB table
- **Files Changed**: `cmd/clonenext.go`, `cmd/clonenexthistory.go`, `cmd/clone.go`, `store/version_history.go`, `model/version_history.go`, `constants/constants_version_history.go`, `store/store.go`
- **Learning**: Default behaviors should optimize for the common workflow; opt-in flags for edge cases
- **Never Repeat**: Do not create accumulating folder structures without a cleanup mechanism
