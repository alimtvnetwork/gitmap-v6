# CD — Navigate to Repository Directory

## Overview

The `cd` command outputs the absolute path of a tracked repository so
the user's shell can change to that directory. Since a child process
cannot change the parent shell's working directory, gitmap prints the
path and the user wraps it in a shell function.

---

## How It Works

1. User runs `gitmap cd <name>` with a repo slug or partial name.
2. Gitmap queries the database for matching `AbsolutePath` entries.
3. **Single match** → print the path to stdout.
4. **Multiple matches** (same repo cloned to several locations) →
   show a numbered list and prompt the user to pick one.
5. **No match** → print an error to stderr and exit 1.

### Shell Integration

Because a subprocess cannot `cd` the parent shell, `gitmap setup`
auto-installs a `gcd` shell function into the user's profile. After
running setup, the user types `gcd myrepo` and the shell changes
directory.

The function is installed to the appropriate profile:
- **PowerShell** → `$PROFILE`
- **Bash** → `~/.bashrc`
- **Zsh** → `~/.zshrc`

The marker `# gitmap shell wrapper v2` prevents duplicate installs.
The wrappers set `GITMAP_WRAPPER=1` before calling the real binary so
gitmap can detect whether the shell function is active. If not, `gitmap cd`
prints a stderr warning advising the user to reload their profile.

**PowerShell (`$PROFILE`):**
```powershell
function gcd { Set-Location (gitmap cd @args) }
```

**Bash/Zsh (`~/.bashrc` / `~/.zshrc`):**
```bash
gcd() { cd "$(gitmap cd "$@")" ; }
```

Gitmap prints only the path to stdout so piping works cleanly.
Prompts and errors go to stderr.

## Commands

### `gitmap cd <name>` (alias: `go`)

Navigate to a tracked repo by slug or partial name.

```bash
gitmap cd myrepo
gitmap go myrepo
```

**Single match output (stdout):**
```
C:\repos\github\myrepo
```

**Multiple match output (stderr prompt + stdout path):**
```
Multiple locations found for "myrepo":

  1  C:\repos\github\myrepo
  2  D:\backup\repos\myrepo

Pick [1-2]:
```

User enters a number; the selected path prints to stdout.

### `gitmap cd repos`

Interactive picker showing all tracked repos as a numbered list.

```bash
gitmap cd repos
gitmap go repos
```

**Output (stderr):**
```
TRACKED REPOS

  1  api-gateway
  2  frontend-app
  3  shared-lib
  4  docs-site

Pick [1-4]:
```

User enters a number; the selected repo's path prints to stdout.

Supports `--group <name>` to filter the list by group.

### `gitmap cd repos --group work`

Show only repos in the specified group.

```bash
gitmap cd repos --group work
```

## Default Location

When a repo has multiple clone locations the user can set a default:

```bash
gitmap cd set-default <name> <path>
gitmap cd clear-default <name>
```

Defaults are stored in `.gitmap/output/data/cd-defaults.json`:

```json
{
  "myrepo": "C:\\repos\\github\\myrepo"
}
```

When a default exists and the repo has multiple matches, gitmap
uses the default without prompting (unless `--pick` is passed).

---

## Flags

| Flag | Description |
|------|-------------|
| `--group <name>` | Filter `repos` list by group |
| `--pick` | Force interactive picker even if a default is set |

## Constraints

- Only the resolved path goes to stdout; all prompts/errors to stderr.
- Slug matching follows the same rules as `gitmap pull` (exact then substring).
- The `repos` keyword is reserved and cannot be used as a repo name.
- All files under 200 lines, all functions 8–15 lines.
